Projects

The Projects module allows you to define and manage scientific projects that will use data from your database. A project can be a machine learning study, a classical statistical analysis, a management dashboard, or any other data-driven initiative.

For each project, you specify which general concepts are required, allowing you to track data requirements and ensure your data collection supports your research objectives.

Projects List

The main Projects page displays all defined projects in a table with the following columns:

  • Project Name — The name of the project
  • Short Description — A brief description of the project's purpose
  • Actions — Edit or delete the project

Creating a Project

Click the Add Project button to create a new project. You'll need to provide:

  • Project Name — A descriptive name for your project
  • Short Description — A brief summary of what the project aims to accomplish

Managing Projects

From the Actions column, you can:

  • Edit — Modify the project name and description
  • Delete — Remove the project from the system

Opening a Project

Double-click on any project row to open the project detail view. This opens a dedicated page where you can configure the project's requirements and documentation.

Project Detail View

When you open a project, you'll see two tabs:

1. Context Tab

This tab provides space for project documentation:

  • Scientific Justification — Describe the scientific rationale for the project, research questions, and expected outcomes
  • Bibliography — List relevant literature and references that inform the project

This documentation helps team members understand the project context and ensures consistent interpretation of data requirements.

Future Enhancements

The project detail page will be progressively enriched with additional fields such as inclusion/exclusion criteria, study population characteristics, and the ability to add custom fields to capture project-specific information.

2. Variables Tab

This tab allows you to specify which general concepts are required for your project. The interface is divided into two panels:

Available Concepts (Left Panel)

Displays all general concepts from the INDICATE dictionary. You can:

  • Select concepts — Click on individual rows to select them
  • Select all / Deselect all — Use the buttons above the table to quickly select or deselect all concepts
  • Add selected concepts — Click the Add selected concepts button to move selected concepts to the project requirements

Selected Concepts for Project (Right Panel)

Shows the general concepts that have been assigned to this project. You can:

  • Select concepts — Click on individual rows to select them
  • Select all / Deselect all — Use the buttons above the table to quickly select or deselect all concepts
  • Remove selected concepts — Click the Remove selected concepts button to remove concepts from the project requirements

Using Project Requirements

Defining project requirements serves two key perspectives:

  • For Project Creators — Identify which data providers can participate in the project based on the availability of required concepts in their databases
  • For Data Providers — Discover which projects they can contribute to based on the concepts available in their data
Best Practices

When defining project requirements, include all general concepts that will be used in your analysis, even if they're only needed for adjustment or stratification. This ensures complete data collection from the start.
