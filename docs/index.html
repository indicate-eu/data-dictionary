Checking database status...
Vocabulary files (CSV or Parquet) are available from ATHENA. Download the desired vocabularies, then select the folder below. The database will be stored locally in your browser and persists across sessions.
Parquet format recommended. With Parquet files, you can browse the hierarchy of any OMOP concept — including concepts not yet in the catalog. With CSV files, hierarchy browsing is limited to concepts already used in existing concept sets. 
# Install: brew install duckdb (macOS) or see duckdb.org/docs/installation

cd /path/to/athena_download

for f in CONCEPT CONCEPT_ANCESTOR CONCEPT_RELATIONSHIP CONCEPT_SYNONYM \
         RELATIONSHIP VOCABULARY DOMAIN CONCEPT_CLASS DRUG_STRENGTH; do
  [ -f "$f.csv" ] && duckdb -c \
    "COPY (SELECT * FROM read_csv('$f.csv', delim='\t', header=true, quote='')) \
     TO '$f.parquet' (FORMAT PARQUET);"
done
# pip install duckdb

cd /path/to/athena_download

python3 -c "
import duckdb, os
for f in ['CONCEPT','CONCEPT_ANCESTOR','CONCEPT_RELATIONSHIP','CONCEPT_SYNONYM',
          'RELATIONSHIP','VOCABULARY','DOMAIN','CONCEPT_CLASS','DRUG_STRENGTH']:
    if os.path.exists(f+'.csv'):
        duckdb.sql(\"COPY (SELECT * FROM read_csv('\"+f+\".csv', delim='\t', header=true, quote='')) TO '\"+f+\".parquet' (FORMAT PARQUET)\")
        print(f'  {f}.csv -> {f}.parquet')
"

Run this in the folder containing the Athena CSV files. Both CSV and Parquet files can coexist — Parquet files are preferred when both are present.
